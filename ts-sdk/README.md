# @opnsense-sdk/client

A fully typed TypeScript client for the OPNsense API, generated from the
introspected OpenAPI spec (`../api-introspect/openapi.json`).

Types come from [`openapi-typescript`](https://openapi-ts.dev); the runtime
client is a thin wrapper over [`openapi-fetch`](https://openapi-ts.dev/openapi-fetch/),
so every path, method, path parameter, request body and response is type-checked
against the spec.

## Setup

```sh
npm install
npm run generate   # ../api-introspect/openapi.json -> src/generated/schema.d.ts
npm run build
```

`src/generated/schema.d.ts` is generated (git-ignored); rerun `npm run generate`
whenever the spec is regenerated by `api-introspect`.

> Requires Node >= 20 (the `docs` command uses `@scalar/cli`, which needs the
> global `File` added in Node 20).

## Usage

```ts
import { createOpnsenseClient } from "@opnsense-sdk/client";

const api = createOpnsenseClient({
  baseUrl: "https://192.168.1.1",
  apiKey: process.env.OPNSENSE_KEY,
  apiSecret: process.env.OPNSENSE_SECRET,
});

// Typed path, params, and response.
const { data, error } = await api.GET("/api/auth/group/get/{uuid}", {
  params: { path: { uuid: "0cc7…" } },
});
if (data) {
  // `data.group` is typed; on proven (non-heuristic) endpoints it is required.
}

// Writes are typed too — the request body shape is enforced.
await api.POST("/api/auth/group/set/{uuid}", {
  params: { path: { uuid: "0cc7…" } },
  body: { group: { name: "admins", description: "…" } },
});
```

OPNsense authenticates with an API key/secret pair sent as HTTP Basic
credentials (key = username, secret = password); `createOpnsenseClient` attaches
them to every request.

### A note on the type fidelity

The spec carries `x-opnsense-direction-source` per operation. Bodies from
`wrapper:*` / `base-action:*` are proven contracts — their top-level key is
marked `required`, so the generated types make it non-optional. `name-heuristic`
bodies are best-effort: every field is optional, so the types never force you to
handle a field that might not be returned. Responses are open-world (a real
response may include more fields than typed).

## Browse the API (Scalar)

```sh
npm run docs   # serves an interactive Scalar API reference, watching the spec
```

This loads `../api-introspect/openapi.json` in
[Scalar](https://github.com/scalar/scalar) and live-reloads as the spec changes.

## Scripts

| Script | Action |
| --- | --- |
| `npm run generate` | Generate `src/generated/schema.d.ts` from the OpenAPI spec |
| `npm run build` | Compile to `dist/` |
| `npm run typecheck` | Type-check without emitting |
| `npm run docs` | Serve the Scalar API reference (Node >= 20) |
